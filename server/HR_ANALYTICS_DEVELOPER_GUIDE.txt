  1. Node.js v18 or higher

     Download: https://www.mongodb.com/try/download/community
     Verify:   mongod --version
     The DB server must be running before you start the app.

  3. Git (optional but recommended)
     Download: https://git-scm.com

  4. VS Code (recommended editor)
     Download: https://code.visualstudio.com

───────────────────────────────────────────────────────────────────────────────
  SECTION 3 — FIRST TIME SETUP (DO THIS ONCE)
───────────────────────────────────────────────────────────────────────────────

  Step 1 — Start MongoDB
  ─────────────────────
  Windows:  Open Services → find "MongoDB" → Start
            OR run:  net start MongoDB

  macOS:    brew services start mongodb-community
  Linux:    sudo systemctl start mongod

  Verify MongoDB is running:
    mongosh
    > show dbs
    > exit

  ─────────────────────────────────────────────────────────────────────────────

  Step 2 — Install backend dependencies
  ──────────────────────────────────────
  Open a terminal, navigate to the server folder:

    cd UKG/server
    npm install

  This installs: Express, Mongoose, Winston, ExcelJS, PDFMake,
  JWT, bcrypt, node-cron, Zod, Helmet, and all other dependencies.

  ─────────────────────────────────────────────────────────────────────────────

  Step 3 — Configure environment variables
  ─────────────────────────────────────────
  The file server/.env already exists with development defaults.
  Open it and verify these values:

    NODE_ENV=development
    PORT=5000
    MONGO_URI=mongodb://localhost:27017/hr_analytics
    JWT_SECRET=your_super_secret_jwt_key_change_in_production
    JWT_EXPIRES_IN=8h
    UKG_API_AVAILABLE=false        ← KEEP THIS FALSE until real API is ready
    LOG_LEVEL=info
    LOG_DIR=./src/logs
    EXPORT_DIR=./src/exports
    BACKUP_DIR=./src/backups

  IMPORTANT: Do NOT change UKG_API_AVAILABLE to true until you have
  real UKG API credentials. The system works perfectly with seeded data.

  ─────────────────────────────────────────────────────────────────────────────

  Step 4 — Run the seed script
  ────────────────────────────
  This creates 25 employees with 90 days of realistic attendance data.
  Run it ONCE. Running again with --reset will wipe and re-seed.

    cd UKG/server

    # First time (creates data):
    npm run seed

    # If you want to wipe everything and start fresh:
    npm run seed:reset

  What the seed script creates:
    ✔ 3 system users (HR, IT Admin, Time Office)
    ✔ 25 employees across all departments and shifts
    ✔ ~2,700 daily attendance records (25 emp × 90 days)
    ✔ ~4,000+ punch records (IN/OUT with realistic jitter + duplicates)
    ✔ Monthly attendance summaries
    ✔ Leave records (1-3 per employee)
    ✔ Holidays within the date range
    ✔ 1 seed job log entry

  Seed output will look like:

    👤  Creating system users...

    👷  Creating employees...
    🏖️   Creating leave records...
    ⏰  Building attendance records...
    📊  Building monthly summaries...
    ✅  Seed complete!

  SECTION 6 — UNDERSTANDING THE CRON JOBS
───────────────────────────────────────────────────────────────────────────────

  Jobs start automatically when the server starts.
  With UKG_API_AVAILABLE=false, every job runs on schedule but logs 'skipped'.
  This is intentional — it keeps job_logs populated with real schedule data.

  ┌────────────────────────────┬─────────────────┬──────────────────────────┐
  │  Job Name                  │  Schedule       │  What it does            │
  ├────────────────────────────┼─────────────────┼──────────────────────────┤
  │  attendance_live_sync      │  Every 5 min    │  Pulls punch records     │
  │  leave_sync                │  Every 30 min   │  Pulls leave requests    │
  │  shift_sync                │  Every hour     │  Pulls shift schedules   │
  │  employee_sync             │  Daily at 01:00 │  Pulls employee master   │
  └────────────────────────────┴─────────────────┴──────────────────────────┘

  Check job status any time:
    GET /api/v1/monitoring/jobs

  SECTION 7 — COMMON ERRORS AND FIXES
───────────────────────────────────────────────────────────────────────────────

  Error: "MongoServerSelectionError: connect ECONNREFUSED"
  Fix:   MongoDB is not running. Start it (see Section 3, Step 1).

  Error: "Cannot find module '...'"
  Fix:   Run npm install in the server/ folder.

  Error: "JWT_SECRET is not set" or token errors
  Fix:   Check that server/.env exists and has JWT_SECRET value.

  Error: Seed fails with "duplicate key error"
  Fix:   Data already exists. Either:
         a) Run npm run seed:reset  (wipes and re-seeds)
         b) Do nothing — existing data is fine

  Error: "EACCES: permission denied" on exports folder
  Fix:   The exports/ folder needs write permission.
         mkdir -p server/src/exports/attendance
         mkdir -p server/src/exports/overtime
         mkdir -p server/src/exports/temp

  Error: Export endpoint returns 404 "No data found"
  Fix:   The date/month/year in your query has no seeded data.
         Use dates within the last 90 days.
         Check today's date and use month/year accordingly.

  Error: 403 Forbidden on a route
  Fix:   Your user role does not have access to that route.
         Use the IT Admin account (admin@steelplant.in) for full access.

  SECTION 9 — WHEN UKG API BECOMES AVAILABLE
───────────────────────────────────────────────────────────────────────────────

  Only 3 files need changes. Nothing else in the system changes.

  1. server/.env
     Change:  UKG_API_AVAILABLE=false
     To:      UKG_API_AVAILABLE=true
     Also set: UKG_BASE_URL, UKG_CLIENT_ID, UKG_CLIENT_SECRET, UKG_COMPANY_ID

  2. server/src/services/sync/ukgClient.js
     The file has TODO comments in each method.
     Implement the axios calls in fetchEmployees(), fetchPunches(),
     fetchLeaves(), fetchShiftSchedules().
     The token caching in getAccessToken() is already wired up.

  3. server/src/services/sync/index.js
     Map the API response fields to your model field names if they differ.
     The upsert logic (hashRecord + hasChanged) is already in place.

  That is all. No controller, route, model, or any other file needs changes.

───────────────────────────────────────────────────────────────────────────────
  SECTION 10 — PRODUCTION CHECKLIST (BEFORE GOING LIVE)
───────────────────────────────────────────────────────────────────────────────

  Security:
    □ Change JWT_SECRET to a long random string (32+ characters)
    □ Change all user passwords from Admin@123 to strong passwords
    □ Set NODE_ENV=production in .env
    □ Restrict CORS to your actual LAN IP range in config/cors.js
    □ Enable firewall on the server — only allow LAN traffic on port 5000

  MongoDB:
    □ Enable MongoDB authentication (create DB user with password)
    □ Update MONGO_URI with credentials:
        mongodb://dbuser:dbpassword@localhost:27017/hr_analytics
    □ Set up automatic daily backup (backup job runs at 01:00 by default)
    □ Test restore from backup before going live

  Server:
    □ Use PM2 to run the server as a background process:
        npm install -g pm2
        cd server
        pm2 start src/server.js --name "hr-analytics"
        pm2 save
        pm2 startup
    □ Set up log rotation (already handled by Winston daily rotate)
    □ Monitor disk space — exports and backups grow over time

  Application:
    □ Verify all 3 user accounts have correct roles
    □ Test all export endpoints and verify Excel files open correctly
    □ Test the login flow with each user role
    □ Confirm job schedule is running (GET /api/v1/monitoring/jobs)
